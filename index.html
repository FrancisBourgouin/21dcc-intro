What is JavaScript?

JavaScript is an interpreted language where either a browser or specific software, like Node.js, will execute.

Where does it come from?
  • JavaScript is a language developed by Brendan Eich in 1995, to add interactivity to the browser. (Remember Netscape?)
  • Microsoft implemented a reverse-engineered version called JScript for Internet Explorer in 1996, which led to support issues between IE and other browsers.
  • A standardized version of JavaScript (ECMAScript) was officialized 1997.
  • In early 2009 ECMAScript 5 was adopted and disparities between browsers was officially over.
  • ECMAScript 2015, also known as ES6, is the version most widely used by everyone right now.

Variable syntax & manipulation

Before going further with our entries, let's look at what we can do with variables.

We have two different kinds of numbers:

  • Integers, also known as round numbers, like ...,-1,0,1,2,3...
  • Floats, numbers that have a decimal point, like 3.14, -6.28

No need for quotes around numbers!

For numbers, we can:

  • Add: 1 + 1
  • Substract: 1 - 1
  • Multiply: 1 * 1
  • Divide: 1 / 1
  • And more!

We assign numbers to variables like this:

let someRoundNumber = 437;
let someFloatNumber = 3.1415926535;

We can put whatever we want in strings:

  • Letters, words & sentences, like 'j', 'Hello' & 'Hey buddy!'
  • Numbers, like '1234', '2.718', '1 4 6 7'
  • Emojis, like 'üòé', 'ü§¶‚Äç', 'ü§∑‚Äç'

WARNING!

As long as there are quotes, doublequotes or backticks, it will be considered a string!

Since strings are not numbers, we can't use substract, divide or multiply operations directly.

However, we can add strings together, also known as concatenation.

"Hello" + "There"
=> "HelloThere"
We assign strings to variables like this:

Comparisons & conditions

For the eagle-eyed, you saw that in an entry we have a small problem. 
"...and true indigenous to Canada."
Should be 'is indigenous', not 'true indigenous'.
That is because the data type for our indigenous variable is a boolean.
A boolean variable has either true, or false as a value like this: 
let beTrue = true;
let beFalse = false;

We can assign a boolean value to a variable.

We can also get a boolean out of a comparison between values.

We can compare using: 
1 === 1   // True, Equal & identic
1 == '1'  // True, Equivalent
1 !== '1' // True, Equivalent value, but different type
1 != '1'  // False, Equivalent
1 > 10    // False, 1 is not bigger, also >= for bigger or equals
5 < 6     // True, 5 is smaller, also <= for smaller or equals
Since we can now compare, it means we can make conditions. "If the tree is indigenous to Canada, return the string 'is', if no, return 'is not'".
In code form: 

let isOrIsNot = "";
if (specificTree.isIndigenous) {
	isOrIsNot = "is";
} else {
	isOrIsNot = "is not";
}
the value will then adjust itself based on the result of the condition.

Encapsulating instructions in functions

Our previous conditional statement wasn't that long, but again, we will need to write it over and over.
With functions, we can execute multiple operations: 

// Declaring the function
const someFunction = function(){
	console.log("Hey!");
	console.log("Ho!");
	console.log("Let's go!");
};

// Calling the function
someFunction();
will output: 

=> Hey!
=> Ho!
=> Let's go!
We can also ask the function to return a value: 

// Declaring the function
const someOtherFunction = function(){
	let a = 20;
	let b = 30;

	return a + b;
};

// Calling the function
let result = someOtherFunction();

console.log(result)
will output: 

=> 50
We can also ask the function to take a parameter: 

// Declaring the function
const someOtherFunction = function(someParameter){
	let a = 20;
	let b = 30;

	return a + b + someParameter;
};

// Calling the function & storing the return value in result
let result = someOtherFunction(50);

console.log(result)
will output: 

=> 100

What if we could execute the function over and over?

automate your life with loops!

With the function we made, we can output one entry.

But we still need to execute the function one at a time.

Loops are very useful for two things:

  • To repeat an instruction a lot of times.
  • To repeat an instruction for a yet undefined amount of times.
This is how a standard for loop looks and operate: 

for (let iterator = 0; iterator < amountOfTimes; iterator++) {
	// Do something
}
If we were to read that in plain english, it would be like:

"For the value of my iterator that starts at 0, as long as it's smaller than amountOfTimes, increment by one my iterator"

The condition can be anything, as long as it's not causing an infinite loop! 

for (let iterator = 0; iterator < amountOfTimes; iterator++) {
	// Do something
}
By using the .length method, we can know how many elements there is in an array!
